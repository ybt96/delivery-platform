# 地方特色局域网络物资配送平台 - 商家联盟与村级推客模块设计

## 1. 模块定位与核心目标
本模块旨在解决本地实体商家单打独斗、缺乏营销手段的问题，同时降低乡亲们的购买决策成本。
核心策略：**“跨店套餐组合” + “村级推客（团长）组局推广”**。

## 2. 角色与权限设计
*   **联盟商家：** 同意加入“百商联盟”的本地商铺，愿意为联盟套餐提供低于散卖的“协议底价”或专属折扣。
*   **村级推客 (团长)：** 熟悉本村情况、有一定威信或人脉的村民（如村里小卖部老板、热心村民）。负责选品组合、在村内微信群推广。
*   **消费者 (村民)：** 通过团长分享的链接或在平台上直接购买套餐的用户。

## 3. 核心业务流程

### 3.1 团长跨店“组局” (套餐创建)
*   **选品池：** 系统汇总所有联盟商家的商品及“协议底价”，形成内部选品池。
*   **自定义组合：** 团长在小程序端，根据本村近期的需求，挑选不同商家的商品组成套餐。
    *   *案例：* 团长发现村里马上办红白喜事，于是把“张三生鲜的猪肉”+“李四杂货的干货调料”+“王五酒水的白酒”组合成【特惠宴席包】。
*   **定价与佣金：** 团长在各商品底价的基础上，设置一个合理的“套餐售价”（通常低于村民自己去各店单买的总价）。套餐售价减去底价总和，即为团长的**预估佣金**。

### 3.2 推广与一键购买
*   团长将生成的【特惠宴席包】分享至本村的微信群。
*   村民点击进入，看到的是一个打包好的商品组合，直接一键支付，无需再分别去各店凑单，享受了实打实的优惠。

### 3.3 统配履约与资金自动清算
*   **履约流转：** 订单生成后，自动接入“配送预约模块”的集散统配逻辑。骑手去各店拿货，打包送至该村。
*   **智能分账：** 订单确认收货后，微信支付分账系统自动进行资金切割：
    1.  **商家货款：** 按协议底价结算给对应供货的张三、李四、王五。
    2.  **团长佣金：** 自动打入该村级推客的账户。
    3.  **配送运费：** 结算给当值骑手。
    4.  **平台服务费：** (如有) 结算给平台方。
