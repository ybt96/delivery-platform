## 一、服务器与基础设施准备

### 1.1 推荐配置（初期低成本方案）

| 项目 | 推荐方案 | 预估费用/月 |
|------|----------|-------------|
| 云服务器 | 腾讯云/阿里云 轻量应用服务器 2核4G | ¥50-80 |
| 数据库 | 服务器自建 MySQL（初期够用） | 包含在服务器内 |
| 域名 | .com 或 .cn | ¥50-80/年 |
| SSL 证书 | 腾讯云免费 DV 证书（1年免费续签） | 免费 |
| 对象存储 | 腾讯云 COS（商品图片存储） | ¥5-10 |
| CDN | 腾讯云 CDN（加速图片访问） | ¥5-10 |
| **合计** | | **约 ¥70-110/月** |

### 1.2 服务器初始化

```bash
# 以 Ubuntu 22.04 LTS 为例

# 1. 系统更新
sudo apt update && sudo apt upgrade -y

# 2. 创建应用用户（不用 root 运行服务）
sudo adduser deploy
sudo usermod -aG sudo deploy

# 3. 安装基础软件
sudo apt install -y git curl wget vim ufw

# 4. 配置防火墙
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 5. 安装 Node.js (v20 LTS)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v  # 验证版本

# 6. 安装 PM2（Node.js 进程管理器）
sudo npm install -g pm2

# 7. 安装 Nginx
sudo apt install -y nginx
sudo systemctl enable nginx
```

## 二、域名与备案

### 2.1 域名注册
- 推荐在腾讯云或阿里云购买域名
- 建议选择：`xianglixtq.com` 或 `xianglixtq.cn`

### 2.2 ICP 备案（必须）
> ⚠️ **中国大陆服务器必须完成 ICP 备案后才能使用域名**
> 微信小程序的服务器域名也必须是已备案域名

- 备案周期：约 7-15 个工作日
- 在云服务商控制台提交备案申请
- 个人备案所需材料：
  - 身份证正反面照片
  - 域名证书
  - 核验照片（部分省份需要）

### 2.3 DNS 解析配置

```
# 在域名控制台添加以下解析记录
A    @           -> 服务器公网IP
A    api         -> 服务器公网IP     # API 接口域名
A    admin       -> 服务器公网IP     # 管理后台域名（未来）
```

## 三、SSL 证书配置（HTTPS）

> 微信小程序要求后端接口**必须使用 HTTPS**

### 3.1 申请免费证书

```bash
# 方案一：腾讯云/阿里云控制台申请免费 DV 证书
# 在 SSL 证书管理页面 → 申请免费证书 → 填写域名 → 验证 → 下载

# 方案二：使用 Let's Encrypt（完全免费，自动续签）
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d api.xianglixtq.com
# 设置自动续签
sudo certbot renew --dry-run
```

### 3.2 证书文件放置

```bash
# 如果是手动下载的证书
sudo mkdir -p /etc/nginx/ssl
sudo cp api.xianglixtq.com.pem /etc/nginx/ssl/
sudo cp api.xianglixtq.com.key /etc/nginx/ssl/
sudo chmod 600 /etc/nginx/ssl/*.key
```

## 四、MySQL 生产数据库部署

### 4.1 安装 MySQL 8.0

```bash
sudo apt install -y mysql-server
sudo systemctl enable mysql
sudo systemctl start mysql

# 安全初始化（设置 root 密码、移除测试库等）
sudo mysql_secure_installation
```

### 4.2 创建生产数据库和专用用户

```sql
-- 以 root 登录 MySQL
sudo mysql -u root -p

-- 创建数据库
CREATE DATABASE xianglixtangqin CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用数据库用户（不要用 root！）
CREATE USER 'xiangli_app'@'localhost' IDENTIFIED BY '你的强密码_至少16位_含特殊字符';
GRANT SELECT, INSERT, UPDATE, DELETE ON xianglixtangqin.* TO 'xiangli_app'@'localhost';
FLUSH PRIVILEGES;
```

### 4.3 导入表结构和初始数据

```bash
# 上传 SQL 文件到服务器
scp docs/sql/init_schema.sql deploy@服务器IP:/home/deploy/
scp docs/sql/seed_data.sql deploy@服务器IP:/home/deploy/

# 在服务器上执行
mysql -u xiangli_app -p xianglixtangqin < /home/deploy/init_schema.sql

# ⚠️ 生产环境不要导入 seed_data.sql（那是测试数据）
# 只需要导入必要的初始化数据（如分类表、村落表）
```

### 4.4 MySQL 性能优化（my.cnf）

```ini
# /etc/mysql/mysql.conf.d/mysqld.cnf 追加

[mysqld]
# 连接数
max_connections = 200

# 缓冲池（设为可用内存的 60-70%，如 2G 服务器设 1.2G）
innodb_buffer_pool_size = 1200M

# 日志
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2

# 字符集
character_set_server = utf8mb4
collation_server = utf8mb4_unicode_ci

# 时区
default_time_zone = '+08:00'
```

## 五、Node.js 后端服务部署

### 5.1 上传代码


### 5.2 配置生产环境 .env

```bash
# /home/deploy/delivery-platform/platform-server/.env

PORT=3000
NODE_ENV=production
LOG_LEVEL=info

# 数据库 —— 使用专用账号，不用 root

DB_PASSWORD=你的强密码_至少16位_含特殊字符

# JWT —— 生产环境必须使用强随机密钥！

# 微信支付

# 腾讯地图
TENCENT_MAP_KEY=你的腾讯地图Key

# 文件上传 —— 生产环境推荐 OSS
UPLOAD_PROVIDER=oss
OSS_BUCKET=xiangli-images
OSS_REGION=ap-chengdu
OSS_ACCESS_KEY=你的OSS_AK
OSS_SECRET_KEY=你的OSS_SK

# 短信服务（上线后启用）
SMS_PROVIDER=aliyun
SMS_ACCESS_KEY=你的短信AK
SMS_SECRET_KEY=你的短信SK
SMS_SIGN_NAME=乡里乡亲
SMS_TEMPLATE_CODE=SMS_123456
```


## 六、Nginx 反向代理配置

### 6.1 API 接口配置

```nginx
# /etc/nginx/sites-available/xiangli-api

server {
    listen 80;
    server_name api.xianglixtq.com;

    # HTTP 强制跳转 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.xianglixtq.com;

    # SSL 证书
    ssl_certificate     /etc/nginx/ssl/api.xianglixtq.com.pem;
    ssl_certificate_key /etc/nginx/ssl/api.xianglixtq.com.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # 安全头
    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options DENY;
    add_header X-XSS-Protection "1; mode=block";

    # 上传文件大小限制（商品图片）
    client_max_body_size 10M;

    # API 反向代理
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源（上传的图片，如果用本地存储）
    location /uploads/ {
        alias /home/deploy/delivery-platform/platform-server/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 健康检查路径不记录日志
    location = /health {
        proxy_pass http://127.0.0.1:3000;
        access_log off;
    }

    # 访问日志
    access_log /var/log/nginx/xiangli-api-access.log;
    error_log  /var/log/nginx/xiangli-api-error.log;
}
```

### 6.2 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/xiangli-api /etc/nginx/sites-enabled/
sudo nginx -t          # 验证配置
sudo systemctl reload nginx
```

## 七、微信小程序审核与上线

### 7.1 前置条件清单

| # | 条件 | 状态 |
|---|------|------|
| 1 | 域名 ICP 备案完成 | □ |
| 2 | HTTPS 证书配置完成 | □ |
| 3 | 后端服务正常运行在 HTTPS 域名下 | □ |
| 4 | 微信公众平台小程序账号注册完成 | □ |
| 5 | 小程序名称、头像、介绍已设置 | □ |
| 6 | 小程序服务类目已选择 | □ |

### 7.2 小程序服务器域名配置

在 [微信公众平台](https://mp.weixin.qq.com/) → 开发管理 → 开发设置 → 服务器域名：

| 类型 | 域名 |
|------|------|
| request 合法域名 | `https://api.xianglixtq.com` |
| uploadFile 合法域名 | `https://api.xianglixtq.com`（如果用服务器上传） |
| downloadFile 合法域名 | `https://img.xianglixtq.com`（图片CDN域名） |

### 7.3 小程序前端修改

```javascript
// miniapp/api/request.js
// 将本地开发地址改为生产域名
const BASE_URL = 'https://api.xianglixtq.com';
```

```json
// miniapp/manifest.json
// 确保 appid 是正式的
{
  "mp-weixin": {
    "appid": "wx你的正式AppID"
  }
}
```

### 7.4 编译与上传

1. HBuilderX → 发行 → 小程序-微信
2. 打开微信开发者工具 → 上传代码
3. 填写版本号（如 `1.0.0`）和项目备注

### 7.5 提交审核

在微信公众平台 → 版本管理：
1. 提交审核（选择类目：生活服务 → 生活）
2. 审核周期：通常 1-3 个工作日
3. 审核通过后 → 点击"发布" → 正式上线

### 7.6 审核常见被拒原因及预防

| 被拒原因 | 预防措施 |
|---------|---------|
| 功能不完整 | 确保所有页面可正常操作，不能有空白页 |
| 测试账号未提供 | 审核备注中提供测试账号和说明 |
| 涉及支付未配置 | 先完成微信支付商户号认证 |
| 个人主体不允许电商类 | 如涉及交易，需注册企业主体 |
| 内容不合规 | 顺风车描述避免出现"出租"、"运营"字样 |

## 八、微信支付接入

### 8.1 注册微信支付商户

> ⚠️ 微信支付需要**企业主体**（个体工商户也可以）

1. 访问 [微信支付商户平台](https://pay.weixin.qq.com/) → 注册
2. 提交营业执照、法人身份证等材料
3. 审核通过后获取：
   - 商户号（mch_id）
   - API 密钥（key）
   - API 证书文件

### 8.2 小程序关联商户号

微信公众平台 → 微信支付 → 关联商户号

### 8.3 后端支付接口开发（后期需要新增）

```
POST /api/order/pay       # 用户发起支付，返回微信支付参数
POST /api/order/pay/notify  # 微信支付回调通知（验签+更新订单状态）
POST /api/order/refund    # 退款接口
```

### 8.4 过渡方案（上线初期）

如果暂未接入微信支付，可以：
- 使用"货到付款"模式：下单 → 配送 → 到付
- 使用"线下转账"：扫商家收款码 → 商家手动确认
- 在订单中标注 `payment_method: 'offline'`

```bash
#!/bin/bash
# /home/deploy/scripts/health-check.sh
# 配合 crontab 每 5 分钟执行一次

HEALTH_URL="https://api.xianglixtq.com/health"
RESPONSE=$(curl -s -o /dev/null -w "%{http_code}" $HEALTH_URL)

if [ "$RESPONSE" != "200" ]; then
  # 发送告警（可以用企业微信机器人或钉钉 webhook）
  curl -s "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的KEY" \
    -H 'Content-Type: application/json' \
    -d '{"msgtype":"text","text":{"content":"⚠️ 乡里乡亲API服务异常！状态码: '$RESPONSE'"}}'
  
  # 尝试自动重启
  pm2 restart xiangli-api
fi
```

## 十、数据备份策略

### 10.1 MySQL 自动备份脚本

```bash
#!/bin/bash
# /home/deploy/scripts/backup-db.sh

BACKUP_DIR="/home/deploy/backups/mysql"
DB_NAME="xianglixtangqin"
DATE=$(date +%Y%m%d_%H%M%S)
KEEP_DAYS=30

mkdir -p $BACKUP_DIR

# 备份
mysqldump -u xiangli_app -p'你的密码' $DB_NAME | gzip > "$BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz"

# 清理超过 30 天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +$KEEP_DAYS -delete

echo "[$DATE] 数据库备份完成" >> /home/deploy/logs/backup.log
```

```bash
# 定时任务：每天凌晨 3 点备份
crontab -e
0 3 * * * /home/deploy/scripts/backup-db.sh
```

### 10.2 备份存储（异地容灾）

```bash
# 将备份同步到腾讯云 COS（推荐）
# 安装 coscli：https://cloud.tencent.com/document/product/436/63144
coscli sync /home/deploy/backups/ cos://xiangli-backup/db-backups/
```

```yaml
name: Deploy to Production
```
