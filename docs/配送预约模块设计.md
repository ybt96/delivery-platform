# 地方特色局域网络物资配送平台 - 配送预约与百货展示模块设计

## 1. 模块定位与核心目标
本模块是平台的核心交易基石，旨在解决乡镇/村落居民购买本地商品“信息不对称”和“取货不便”的痛点。
核心策略：**“商品导向浏览” + “平台集散统配” + “固定班次预约”**。

## 2. 前端交互与浏览选购设计 (用户端)

### 2.1 商品导向界面 (类似淘宝模式)
*   **品类检索入口：** 首页按高频本地生活刚需进行分类（如：新鲜蔬菜、五金建材、日常调料、农资工具等）。
*   **商品聚合展示：** 用户点击某一品类或搜索具体商品（如“大白菜”）时，系统以商品为核心展示列表。
    *   **比价效应：** 列表同屏展示本地多家售卖该商品的商铺报价（如“张三生鲜：1.5元/斤”、“李四菜铺：1.2元/斤”）。
    *   **质量透明：** 强制要求商家上传实拍图，并展示同村居民的真实购买评价（知道好赖）。

### 2.2 跨店全局购物车
*   **无缝合单：** 打破传统单店外卖的限制，允许用户将“张三家的白菜”、“李四家的酱油”和“王五家的螺丝钉”同时加入全局购物车。
*   **统一结算：** 购物车支持跨店商品一键合并结算，简化乡亲们的操作流程，降低使用门槛。

## 3. 预约下单与运费调度设计

### 3.1 班次预约机制 (固定时段配送)
*   为了最大化降低物流成本并契合乡镇生活节奏，摒弃“随时下单随时送”的高成本模式。
*   **引入班次概念：** 
    *   **上午班次：** 每日 10:00 截单，11:00 - 12:00 集中配送到村。
    *   **下午班次：** 每日 16:00 截单，17:00 - 18:00 集中配送到村。
*   用户在结算页必须选择期望的配送班次，系统根据当前时间自动禁用已截止的班次。

### 3.2 统一运费一口价
*   **按距离/村落计费：** 无论用户的合并订单跨了多少家店铺，平台仅向用户收取**一笔总运费**。
*   **定价策略：** 运费系统根据“收货人所在村落”到“镇中心商业集散地”的固定距离进行梯级定价（如：邻村3元/单，偏远村5元/单）。

## 4. 后端订单流转与集散处理逻辑

### 4.1 总单拆分子单与备货
*   **支付与拆单：** 用户完成微信支付（总金额包含商品总价+一笔总运费）后，后端系统自动将总订单拆分为两部分：
    1.  **商家备货子单：** 分发给对应的张三、李四等商家，通知其在规定截单时间前将货物打包备好。
    2.  **集散配送任务单：** 生成一条包含所有取货点和最终送货点的综合配送任务。

### 4.2 骑手/顺路人取货与统配
*   **任务领取：** 平台的专职配送员或接单的“顺路人”（结合顺风车模块）接取该班次的配送任务。
*   **集散动线：** 骑手到达镇中心后，按照系统规划的最佳路线，依次前往张三店取白菜、李四店取酱油。
*   **统一派送：** 取齐所有商品后，集中打包，一次性配送至村里的用户手中。

### 4.3 资金清算与分账 (微信支付分账)
*   **资金暂存：** 用户支付的资金先进入平台关联的微信支付商户号（或利用微信的分账能力暂冻结）。
*   **订单完结触发：** 当用户确认收货（或骑手妥投后24小时自动确认）后，系统触发分账指令。
*   **按比例结算：** 
    *   商品货款（扣除可能的平台微小抽佣后）分别结算至对应商家的微信零钱/银行卡。
    *   运费结算至当值骑手/顺路人的账户。
