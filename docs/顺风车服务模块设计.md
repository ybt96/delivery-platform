# 地方特色局域网络物资配送平台 - 熟人顺风车互助模块设计

## 1. 模块定位与安全原则
区别于传统的商业网约车，本模块定位为**“纯熟人社会的公益/半公益互助出行”**。
核心原则：**极度安全、严格隔离、场景化互助**。主要服务于老人看病、小孩上下学等高信任要求场景。

## 2. 严格的村级隔离准入机制
*   **“出行圈”概念：** 以“行政村”或“自然村落”为单位，在平台上划分出相互独立的“出行圈”。
*   **管理员审批制：** 
    *   每个村设立一名或多名管理员（通常为村长、村委干部或极具威望的村民）。
    *   新用户若想使用顺风车功能（无论是发单还是接单），必须提交真实姓名和关系说明，并由**本村管理员手动审核通过**后，方可加入本村的“出行圈”。
*   **数据绝对隔离：** A村的用户绝对无法看到B村的顺风车行程信息，保证信息只在“知根知底”的范围内流转。

## 3. 核心功能设计

### 3.1 场景化行程发布
*   发单界面不强调单纯的“起点到终点”，而是提供**高频场景模板**：
    *   🟢 【老人就医】需要宽敞座位，司机需有耐心。
    *   🟢 【儿童接送】上下学同路捎带。
    *   🟢 【镇上赶集】买东西顺路带人/带货。
*   发布时可明确备注同行人数及特殊需求（如：轮椅、大件农具）。

### 3.2 费用分摊与互助机制
*   **非营利导向：** 平台不建议按公里数精准计费，不从中抽水。
*   **灵活酬谢：** 
    *   支持设定固定金额的“油费分摊”（如去镇上统收5元/人）。
    *   支持“互助红包”模式，乘客到达后在小程序内发个随喜红包感谢车主。
    *   甚至支持“0元互助”，纯靠乡邻感情。

### 3.3 安全与紧急保障体系
*   **全实名展示：** 圈内展示用户的真实姓名及村内关系（如“村东头老李家儿子”），利用熟人社会的道德约束力保障行为规范。
*   **一键紧急联络：** 行程进行中，提供一键拨打村长/管理员电话，或一键将当前位置发送给紧急联系人的功能。
